## What This Is

`@gfazioli/mantine-list-view-table` — a Finder-style list view table component for Mantine 8.x. Yarn 4 monorepo with two workspaces: `package/` (npm library) and `docs/` (Next.js 15 static site).

Peer dependencies: Mantine (core/hooks) >=8.0.0, React 18/19, @tabler/icons-react ^3.34.0.

## Architecture

### Build Pipeline

- **Input:** `package/src/index.ts` → Rollup → dual ESM (`.mjs`) + CJS (`.cjs`) output
- **CSS Modules:** Scoped with `me-` prefix via `hash-css-selector`
- **`'use client'`** directive auto-injected on all chunks except index (Next.js App Router compat)
- **Type declarations:** Generated by `scripts/generate-dts` into `package/dist/types/`

### Component Pattern

Single-file component (`ListViewTable.tsx`, ~1200 lines). Uses Mantine's `factory` pattern:

```
factory<ListViewTableFactory> → useProps → useStyles → varsResolver
```

**Key design decisions:**
- **Controlled/uncontrolled dual mode** for sorting, column reordering, and column resizing — if no `onSort`/`onColumnReorder` callback is provided, the component manages state internally
- **Column widths** are measured from the DOM on first render, then tracked in state for resize constraints
- **Sticky columns** use inline `position: sticky` with z-index layering (11 for header, 10 for body cells)
- `getNestedValue()` supports dot-notation keys for nested data access

### Styles API

All visual parts are targetable via `stylesNames`: `root`, `table`, `header`, `headerCell`, `headerButton`, `headerTitle`, `sortIcon`, `dragHandle`, `resizeHandle`, `body`, `row`, `cell`, `emptyState`, `loader`, `stickyColumn`, `stickyHeaderColumn`.

CSS variables on root: `--list-view-height`, `--list-view-width`, `--list-view-header-title-font-size`, `--list-view-header-title-font-weight`, `--list-view-cell-font-size`, `--list-view-cell-font-weight`.

### Demo System

Demos live in `docs/demos/` and follow `@mantinex/demo` conventions:
- Named exports (not default), typed as `MantineDemo`
- Dual export pattern: actual data/columns + code string for display
- File naming: `ListViewTable.demo.<feature>.tsx`
- Configurator demos use `{{props}}` placeholder in code templates

## Testing

- Jest 29 + jsdom + `esbuild-jest` transform
- Use `@mantine-tests/core` render utilities (not raw RTL)
- CSS modules mocked with `identity-obj-proxy`
- Test file: `package/src/ListViewTable.test.tsx`

## Adding a New Prop

1. Add to `ListViewTableBaseProps` interface (with JSDoc)
2. Add default in `defaultProps` if applicable
3. Destructure in the component and implement
4. Run `yarn docgen` to regenerate API docs
5. Create demo in `docs/demos/`
6. Add test coverage
